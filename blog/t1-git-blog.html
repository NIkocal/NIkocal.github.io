Version Control

Version control is a system of tracking changes made to files over the span of a project. This is accomplished by taking "snapshots" everytime something is editted in a file. If at any point in time I make a mistake and would like to go back, i can easily do this with and start right before the point i made the mistake. I haven't worked in a team setting before but i can imagine how important it is for all members to have a solid understanding of version control to ensure that all the code is functional and everyone is synchronized.

Git is a distributed version control that runs on your local machine. Git has three main states that files fall in: committed,modified and staged. Committed means that you have made your changes to the file and it is saved in your local repo and ready to pushed to the master. Modified means a files is changed but not committed to your repos yet. Staged means that you have made your changes and that file is ready for your next commit. You can then view your commmit history by using the git log command.

Github is a collaborative platform where you can store your code online. It is an extension of git and allows users to access there there code remotely from anywhere. Each user creates a user profile which displays contact information,showcases your work and all the others you've contributed to. Because of the social network aspect, it makes it easier for users to contribute their skills and expertise to move projects forward.
