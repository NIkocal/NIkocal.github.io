Variable Scope

So lets jump right into it when writing and organizing your code, you will have to put some thought into which variables are accessible in particular locations of your program, this is where scoping comes in. Scoping is a term which defines whether your variable is accessible or not. If done properly you will avoid alot of errors and your code will run as expected . It will also prevent you from namespacing issues and also other methods from accessing or modifying your variables, unless specified .Here are the different variables and their scope that you will find in Ruby : local variable, instance variable, class variable, global variable and constant.

Instance Variables and Class Variables

Class variables are variables that are shared across all instances of a class and are prepended by tow @@ signs . Instance variables has a name beginning with only one @ sign. They are used to store the state of an object and are only visible to the object in which they belong to. Here is a small example of both in action.


class Protein
  @@yummy = true
    attr_accessor :name 
    def initialize(name)
    @name = name
    end
    
    def yummy?
    @@yummy = true
    end
end

c = Protein.new("Jerk-chicken")
f = Protein.new("Snapper")
p c.name =>  "Jerl-chicken"
p f.name  => "Snapper"
p c.yummy? => "true"
p f.yummy? => "true"
