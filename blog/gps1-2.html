Pair Programming

by : Nikolai Calder and Emmanuel Kaunitz

What is Pair Programming

The process of pairing is done when two developers collaborate on a task .This is done by assigning a role to each where one controls the actual writing of code and the other gives instructions, controls the overall structure of the program.

Why is Pair Programming important to DBC ?

Pairing is important to DBC because it creates a collaborative learning environment where each student can actively bounce ideas off their partner and create solutions they wouldn't be able to do on their own. This also prepares students for future interactions when working in a team setting.

Compare driver and navigator

A driver is the one that writes the code and is in control of the keyboard while the navigator instructs the driver and provides insight therefore reducing errors.
