Ruby Classes

Ruby is an object-oriented programming languge which means it manipulates constructs called objects. All of these objects belong to particular classes that define their state and behaviour. The state keeps track of the attributes of objects and behaviors are what objects are capable of doing. Classes allow you to create a blueprint for your objects with uniqe states but similar behaviour. Ruby has built in class but lets create our own player class. 

class Player
end

p = Player.new

Initializaton and Instance variables

This is the most basic way to define a class in Ruby but we haven't but our object is empty. it has no state or behavior . We will define an initialize method that will create each instance with a unique state. Inside the initialize method we store the object's state in instance variables. the values assigned to those instance variables are unique to each object. Instance variables start with an @ sign. Unlike other variables that disappear at the end of an object these will 

class Player

  def initialize(name, age)
    @name = name.capitalize
    @age = age
  end

end
 
 p = Player.new("Steve", 35)

Instance methods

Now that we have a class and instance variables to hold the state. We can create methods that operate on an objects state. To do this we will define the method within the class and it will be shared by all objects of the Person class.

class Player

  def initialize(name, age)
    @name = name.capitalize
    @age = age
  end
  
  def greeting
    puts "I'm #{@name} and i am #{@age} years old"  
  end
end 

p = Player.new("Steve", 35)
n = Player.new("John", 40)
n.greeting => I'm John and i am 40 years old
p.greeting => I'm Steve and i am 35 years old

Wrap up

A whole book can be written on this topic but this is just a quick intro in to creating a simple template for objects,some points to keep in mind are that classes allow you to reduce duplicity in your code and makes it easier to make changes to the program without having to start from scratch.
